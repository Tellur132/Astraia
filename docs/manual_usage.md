# マニュアルの使い方と運用ルール

このページは Astraia マニュアルの「読み方」と「書き方」をまとめたものです。新しいドキュメントを追加する前に必ず確認し、GitHub Pages で閲覧しやすい形を維持してください。

## ディレクトリ構成（`docs/`）
- `index.md`: GitHub Pages のトップページ。全リンクの起点。
- `manual_usage.md`: 本ページ。ルールとテンプレート。
- `guides/`: 手順書・実行ガイド・運用ノウハウ。
- `plans/`: 計画書、ロードマップ、背景資料。

## 読み方のヒント
- 手順を知りたいときは `guides/`、背景や方針を知りたいときは `plans/` を参照。
- ブラウザの検索（`Cmd/Ctrl + F`）で目的のセクションを素早く探す。各ページは `前提` や `完了条件` などの見出しを揃える。
- 章末のリンクで関連ドキュメントへ移動できるよう、相互リンクを追加することを推奨。

## 執筆ルール
- **ファイル名**: ASCII のみ。`kebab-case` または `snake_case`、拡張子は `.md`。スペースや全角文字を避ける。
- **見出し構造**: 先頭に H1（タイトル）、続けて「このページでわかること」「前提」「手順/決定事項」の順で書くと読みやすい。
- **コード/コマンド**: フェンスコードに言語指定（例: ```bash）。変数は `<path>` のようにプレースホルダ表記にする。
- **リンク**: 可能な限り相対パスを使用し、`docs/index.md` から辿れるようにする。新規ページを追加したら index に必ず追記する。
- **言語**: プロジェクト共通で日本語を優先。外部 API 名など固有名詞は英語表記を維持する。
- **更新メモ**: 大きな変更を加えた場合はページ末尾に `Changelog` セクションを追加し、日付 + 1 行で記録する。

## GitHub Pages 運用メモ
- Pages のソースは `docs/` 直下を指定すると本マニュアルがそのまま公開されます（ブランチは `main` を想定）。
- 相対リンクはローカル閲覧と Pages の両方で動くように記述する（先頭 `/` を付けない）。
- サイドバーがないテーマでも迷わないよう、各ページの冒頭に「このページでわかること」など短い導入を書いておく。

## 新規ページを作るときのテンプレ

```markdown
# タイトル

このページでわかること:
- 何を達成できるか（例: GUI バックエンドの起動と主要 API の確認）
- 対象読者（例: ローカル環境で実験を回したい開発者）

## 前提
- 必須の依存や環境変数
- 想定ディレクトリ/ポート など

## 手順 or 決定事項
1. ステップ 1
2. ステップ 2

## 補足 / トラブルシュート
- よくある失敗と回避策
```

> 上記テンプレを基に、内容に応じてセクション名は調整して構いません。
